# Pathgrid — Documentation Pack for a roadmap.sh-Style Learning Platform

Research and specification documents for building an AI-native developer learning platform
functionally inspired by roadmap.sh, with all content generated by Claude/GPT instead of a
human community.

**Product name: Pathgrid** (chosen; collision-checked via web search Jul 2026 — nothing
significant surfaced). Before public launch, still secure the domain (`pathgrid.dev` /
`pathgrid.app` preferred), the @pathgrid handles (GitHub, X), and run a quick EUIPO
trademark search.

---

## How to use this pack

### Step 1 — Review and adjust scope (you)
Read `01-research-findings.md` and `02-product-spec.md`. Trim the P0 scope if you want a
faster first launch. Everything downstream depends on these two documents.

### Step 2 — Design with Claude Design
Feed `07-design-brief.md` plus `03-information-architecture.md` into Claude Design.
Ask it to produce, in this order:
1. A design system (tokens: colors, typography, spacing, components)
2. The Home page
3. The Roadmap page (canvas + topic drawer) — the most important screen
4. AI Hub / chat, Dashboard, Settings (API keys), Auth

Export the token values and screen designs. Save them into the project repo under
`/docs/design/` so Claude Code can reference them.

### Step 3 — Build with Claude Code
Start a repo, copy this whole pack into `/docs/`, and use `08-claude-code-plan.md`:
it contains a phased build plan, a ready-to-paste starter `CLAUDE.md`, and suggested
prompts for each phase. Build phase by phase; don't ask for the whole site at once.

### Step 4 — Generate content
Use `06-ai-content-system.md`. It contains the seeding pipeline and copy-paste prompts
for generating roadmap graphs, topic content, quizzes, and guides with any provider
(Claude, GPT, Gemini, or anything on OpenRouter).
Generate 2–3 roadmaps first, review quality, then batch the rest.

## Four decisions already baked in (change them consciously)

1. **Build from scratch, don't fork.** roadmap.sh's repository is public but under a
   restrictive proprietary license (personal use only; no republishing of content, images,
   or project files). Details in `01-research-findings.md` → "License and IP constraints".
   All code, content, branding, and roadmap data in your build must be original /
   AI-generated. Feature ideas and general layout patterns are fair game.

2. **AI-generated content is the product's core differentiator, not an add-on.** The spec
   treats the content pipeline (doc 06) as a first-class system with schema validation and
   a review step — not ad-hoc prompting.

3. **Recommended stack: Next.js 15 + TypeScript + Tailwind + Postgres + React Flow +
   Vercel AI SDK.** Rationale and an Astro alternative are in `05-tech-architecture.md`.

4. **No monetization, no teams — bring-your-own-key (BYOK) instead.** The app is for
   personal use and free distribution. Every AI feature runs on the user's own API key
   (OpenAI, Anthropic, Google Gemini, or OpenRouter), configured in Settings and kept
   client-side. No pricing pages, plans, meters, or team features exist anywhere in
   the spec.

## Suggested MVP cut (if you want to ship in ~2 weekends)

Home + catalog → interactive roadmap viewer with topic drawer → progress tracking with
local-storage-first, accounts later → 5 seeded AI-generated roadmaps. Everything else
(chat, quizzes, editor, distribution packaging) comes after. See phasing in `02-product-spec.md`.
